# シリーズ再設計: 非エンジニア向けAIコーディング入門

## 背景・課題

PR #4 のレビューで以下の問題が指摘された:

- 「claude code とは」を2026年に検索する人はエンジニアであり、公式サイトやQiita/Zennに行く
- programming-zero.net の読者層（プログラミング初級者・GASラボ顧客）とズレている
- シリーズの目的（非エンジニアへのClaude Code導入コンサル・研修のリード獲得）に対してターゲット設計が不適切

## 決定事項

- 現シリーズ「Claude Code完全ガイド」（エンジニア向け16記事）は **廃止**
- PR #4 はクローズ、WP下書き（postId: 6990）も削除
- 非エンジニア向けの新シリーズに一本化

## コンセプト

```
旧: 「Claude Codeの使い方を教える」（ツール起点・エンジニア向け）
新: 「非エンジニアがAIで自走する方法を教える」（課題起点・非エンジニア向け）
```

## ターゲット

| 項目 | 定義 |
|------|------|
| 誰 | GASラボ顧客層（SNS運用者、マーケター、広告代理店、Web制作会社、店舗、EC事業者） |
| 前提知識 | プログラミング初級者。GASツールは使ったことがある。ターミナル操作は不慣れ |
| 悩み | エンジニアに頼まないとできないことがある。外注コストが高い。対応が遅い |
| ゴール | エンジニアなしでClaude Codeを使って業務課題を解決できる |
| CTA | Claude Code導入コンサル・研修の問い合わせ（LP・フォームは並行して作成） |

## 表現方針

- **図解を多用**: 非エンジニアにとって文字だけでは伝わりにくい概念（エージェント型AI、MCPの仕組み等）は図解で説明
- `/illustrate` スキル（Gemini CLI + Nanobanana）を活用してフロー図・概念図・比較図を生成
- スクリーンショットは実際の操作画面を `/screenshot` で撮影
- 専門用語は初出時に必ず平易な言い換えを添える
